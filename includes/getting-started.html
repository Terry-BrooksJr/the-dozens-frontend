Getting Started With The Yo' Momma Jokes API

Our Yo Mama Jokes API is designed with developers in mind, offering an intuitive and straightforward interface. With well-documented endpoints and clear error messages, you'll be able to integrate our API into your applications seamlessly. Whether you're new to API consumption or a seasoned developer, you'll find our endpoints easy to navigate and implement.

You can make simple HTTP requests to retrieve jokes, with filters for categories and content preferences. Rate limiting ensures the API remains responsive, allowing you to focus on delivering a great user experience without worrying about backend performance.

Step 1: Create Account

Creating a User is required if you plan on contributing to the API. Just looking to laugh, skip to step 2

Endpoint:

POST /auth/users/

Request Body:

{
  "email": "user@example.com",
  "username": "dozens_lover",
  "password": "securePassword123"
}

Expected Response:

{
  "id": 12,
  "username": "dozens_lover",
  "email": "user@example.com"
}

Step 2: Get Random Joke

Endpoint:

GET /api/insults/random/

Headers:

Content-Type: application/json

Optional Query Parameters:

?category=P&nsfw=false

Response:

{
  "reference_id": "GIGGLE_ABC123",
  "content": "Yo mama is so poor, she can't even afford to pay attention.",
  "category": "Poor",
  "nsfw": false,
  "added_by": "John D.",
  "added_on": "2 days ago"
}

In this example, you can specify the joke category and whether NSFW content is allowed. If no filters are provided, a random joke will be returned. Check out our comprehensive documentation for all available endpoints and parameters.
